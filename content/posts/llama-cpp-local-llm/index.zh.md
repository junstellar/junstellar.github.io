如今没人不用Claude或ChatGPT。但如果这类AI能**不联网、不花钱，直接跑在自己的笔记本上**呢? 你也许会想「那不得有一屋子GPU的服务器机房?」——先说结论：**家里一台游戏笔记本，一分钱不花**就能搞定。上次我用ComfyUI搭了个「画画的本地服务器」，这次是用`llama.cpp`把**「会说话、会写字的LLM」**装进自己GPU的实战记录。

## 为什么非要本地跑?

老实说，单论聪明程度，云端AI更强。但本地运行仍有明确的理由。

最大的一点是**可以无限用token**。云端API对每一个输入输出token都要收费。要是把几百份文档做摘要、把几万行日志分类、或者通宵跑点什么，账单就会悄悄地涨得吓人。本地除了电费之外是0元，所以你永远不会因为「这也太费token了吧…」而犹豫。想把提示写多长就写多长，不满意就反复重跑，都随你。

在此之上还有这些好处：

- **没有速率限制。** API对每分钟请求数设上限，大批量任务就得一直等；而我的GPU完全属于我，不用排队。
- **数据不出本机。** 处理公司内部文档、个人信息、还没公开的代码这类不方便上传的东西时，这一点至关重要。压根不上传，也就无从泄露。
- **不需要联网。** 飞机上也好，断网的隔离内网也好，照样能跑。
- **模型不会突然消失。** 云端模型某天可能就把旧版本下线了，而落到自己硬盘上的文件，永远还是那个版本。

一句话概括：**聪明看云端，自由度、成本、隐私看本地。** 数据越敏感、重复性工作越多，本地就越发光。

## 本地LLM，用什么来跑? — 三个候选

跑本地LLM有好几种方式，我比较了三个然后选定了一个。

| 工具 | 优点 | 不足 |
|------|------|-----------|
| **Ollama** | 一行命令就起来，最简单 | 内核终究是llama.cpp，精细调优隔了一层 |
| **LM Studio** | 漂亮的GUI，点几下就行 | 让程序调用来做自动化，偏重 |
| **llama.cpp ← 选它** | **引擎本身**，压榨VRAM最强，OpenAI兼容API | 得稍微熟悉一点命令 |

其实上面的Ollama和LM Studio，内部真正在跑的**引擎正是llama.cpp**。它能让你手动调到往GPU上放多少层，最适合在8GB笔记本上压榨；自带的`llama-server`还会开出**与OpenAI完全相同格式的API**，任何应用都能干净地接上。

## 整体图景 — 原理很简单

原理和ComfyUI那次一样：繁重的计算交给本地GPU，外部程序只管用API去调。

```
用户/应用: 「帮我总结这段」
   ↓
llama-server (本地, http://127.0.0.1:8080/v1)  ← OpenAI兼容API
   ↓
GGUF模型在RTX 4060上逐个生成token
   ↓
答案以流式方式吐出来
```

只需记住两个词。**llama.cpp**是用C/C++写的LLM推理引擎，就算GPU羸弱，也能把模型分摊到CPU和GPU上来跑。**GGUF**是llama.cpp专用的模型格式，是把原始模型**量化**后、大幅削减体积和VRAM占用的文件。

## 我的笔记本行不行? — 先看配置

先看看到底可不可行。和上一篇ComfyUI用的是同一台HP Victus 16笔记本。

| 项目 | 所需配置 (7〜8B模型) | 我的笔记本 | 判定 |
|---|---|---|---|
| GPU显存 | 6GB以上 (Q4) | RTX 4060 Laptop 8GB | 通过 |
| 内存 | 16GB | 16GB | 通过 |
| 硬盘 | 每个模型约5GB | 空间充足 | 通过 |

显存8GB的话，**7〜8B级模型量化到Q4就能整个装进GPU**，这一档最快。从13〜14B级开始就得把一部分层挤到内存里，速度会陡降；30B以上对这台笔记本来说就有点贪心了。

## 挑模型 — 「量化」占一半

即便是同一个模型，**量化(quantization)**方式不同，体积、速度、质量都会变。我在这儿踩了第一个坑。想着「反正要高质量」就下了Q8，结果碰上**显存不够、模型只有一半装进GPU**的情况。最后8GB的现实答案是**`Q4_K_M`**。

| 量化 | 7B模型体积 | 特点 |
|---|---|---|
| Q8_0 | 约7.5GB | 最接近原版，但对8GB偏重 |
| **Q4_K_M** | **约4.5GB** | **速度与质量平衡，本地事实上的标准** |
| Q3_K_M | 约3.5GB | 更轻，但质量下降开始能感觉到 |

模型选7〜8B级的instruct系列(如Qwen2.5-7B-Instruct、Llama-3.1-8B-Instruct 等)都稳妥。GGUF文件通常能在Hugging Face上像`bartowski`这样的发布方按量化档位下载。

## 开始安装 (以Windows 11为例)

### 1. 获取llama.cpp二进制

从源码自己编译也行，但最快的路是下**预编译的CUDA二进制**。在[GitHub发布页](https://github.com/ggml-org/llama.cpp/releases)下载类似`llama-bxxxx-bin-win-cuda-x64.zip`的文件，解压到`C:\llama`就完成了。

### 2. 下载GGUF模型

```powershell
# 7B级 Q4_K_M 模型 (约4.5GB) → 保存到 C:\llama\models\
curl.exe -L -o C:\llama\models\model-q4_k_m.gguf ^
  "https://huggingface.co/bartowski/Qwen2.5-7B-Instruct-GGUF/resolve/main/Qwen2.5-7B-Instruct-Q4_K_M.gguf"
```

### 3. 启动服务器

这里只需记住一个：**`-ngl`** —— 放到GPU上的层数。`-ngl 99`意思是「能放多少放多少」，7〜8B Q4模型会整个进到8GB里。

```powershell
C:\llama\llama-server.exe -m C:\llama\models\model-q4_k_m.gguf -ngl 99 -c 8192 --port 8080
```

- `-ngl 99` : 把全部层放到GPU (显存不够就把数字调小，只放一部分)
- `-c 8192` : 上下文长度8k token
- `--port 8080` : 服务器端口

### 4. 确认能跑起来

在浏览器打开`http://127.0.0.1:8080`，聊天UI会立刻出现。想确认是不是真在用GPU，就看启动日志里`offloaded 29/29 layers to GPU`这样的行。这个数字拉满，就成了。

### 三个常见的坑 (先提个醒)

安装本身很简单，但大家偏偏就在这三处栽跟头。

- **坑1 — 下成了CPU版。** 发布页文件有好几个，一不留神下了`win-cpu`版，GPU就闲着、回答慢得像蜗牛。**NVIDIA一定要下`cuda`版。**(AMD下`vulkan`版)
- **坑2 — 忘了加`-ngl`。** 漏了这个参数，模型就全在CPU上跑，慢到爆。要是你纳闷「明明有GPU，怎么这么慢?」，十有八九就是它。
- **坑3 — 贪心上大模型、高量化。** 就是上面踩的那个坑。一超显存，速度就崩。8GB的话，7〜8B配Q4才是正解。

## 接到应用上 — OpenAI兼容API

`llama-server`真正方便的地方，是它开出的是**和OpenAI完全相同格式的API**。原本用OpenAI的代码，**只要把地址改成本地的**，就能原样跑起来。

```python
from openai import OpenAI
client = OpenAI(base_url="http://127.0.0.1:8080/v1", api_key="local")

resp = client.chat.completions.create(
    model="local",
    messages=[{"role": "user", "content": "你好，做个自我介绍"}],
)
print(resp.choices[0].message.content)
```

就这一行，一个能**离线、免费**跑文档摘要、代码补全、翻译的专属后端就搭好了。

## 到底有多快?

- 7〜8B Q4_K_M全层上GPU的情况下，大约**每秒40〜60个token**，快到眼睛都跟不上。
- 首次响应在几秒内，含把模型装进显存的时间。
- 把上下文拉长(比如16k)可能会显存吃紧，这时把`-ngl`数字调小、往内存里挤一部分层就行。稍微慢一点，但能跑。

## 老实说的结论 — 哪里行，哪里不行

- **行:** 免费、不联网，就在自己笔记本上跑起一个相当能用的LLM。摘要、翻译、分类、写代码辅助这类实务，7〜8B就够。最重要的是能**不心疼token、放开手脚地**用。
- **局限:** 再好的本地模型，也没有顶级云端模型(比如Claude)那么聪明。复杂推理、长代码设计，目前还是云端更强。
- **现实的搭配:** 所以我觉得最好的方式是**敏感数据、大批量重复工作交给本地，难题交给云端**。两者不是竞争，而是分工。

不花钱、用一台笔记本就能做到这一步，本身就说明AI的门槛已经降得多低了。下一步，我打算把这个本地LLM接到像Claude Code这样的工具上，做一个**完全离线的编程搭档**。
