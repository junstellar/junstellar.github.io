> 💻 **インストール:** `pipx install git+https://github.com/junstellar/redmine-mcp-jun.git`

Claude Code から社内 Redmine を直接読み書きできるようにしたかった。ところが、うちの Redmine は社内ネットワークの中にあり、Atlassian や GitHub のような外部 SaaS コネクタが使えない。そこで、自分の PC の中だけで動くローカル MCP サーバーを自作した。単一ファイルから始めて pip パッケージに育てるまでの、一週間の記録だ。

## MCP とは何か、ツールをどう設計したか

MCP（Model Context Protocol）は「AI と外部ツールをつなぐ」標準だ。USB のように、標準さえ合わせればどんなツールも LLM に挿して使える。

サーバーをクライアントにつなぐ方法は大きく二つある。**stdio** はサーバーをユーザーの PC でローカルプロセスとして起動し、その PC の中だけでやり取りする方式。**HTTP（リモート）** はサーバーを別に立ててネットワーク越しにつなぐ方式だ。リモートは一度立てれば複数人で共有しやすいが、サーバー運用や認証・キー管理を自分で面倒みる必要がある。自分は社内で自分用にインストールして使うので、キーが PC の外に出ず、インストールも簡単な **stdio 方式**にした。

Redmine 連携を作るとき、コードより先にやったのは「LLM にこれで何ができるべきか」を書き出すことだった。書き出してみると、普段 Redmine でやることはそう多くない——自分の課題の確認、課題の検索・閲覧、課題の作成、コメント、Wiki の閲覧・編集くらい。それに合わせてツールを 10 個に絞った。

| ツール | 役割 |
|---|---|
| `list_projects` / `list_issues` / `get_issue` | プロジェクト・課題の閲覧 |
| `create_issue` / `add_comment` | 課題作成・コメント |
| `list_wiki_pages` / `get_wiki` / `update_wiki` | Wiki |
| `get_my_today` | 「今日の自分の課題」ショートカット |
| `list_enumerations` | 優先度・ステータス ID |

ここで一つ学んだ。`get_my_today` のように頻繁に使う流れは専用ツールに切り出しておくと、LLM がずっと的確に選んでくれる。そして作りながら身についた三つ——ツールの説明（description）は手厚く（雑だと違うツールを呼ぶ）、入力スキーマは厳密に、レスポンスは JSON をそのまま渡す。

## 本当に大変なのはコードではなく配布だった

最初はファイル一つ（約 420 行）でとりあえず動作を確認した。実際に時間を食ったのは「これを他人の PC にどう入れるか」だった。OS ごとのパス分岐が増え、さらに Python 3.12 の PEP 668 で一部環境では `pip install` 自体が弾かれた。結局、ちゃんとした pip パッケージに作り直して pipx でインストールするようにしたら、どの OS でも二行で済むようになった。

一番気を使ったのは Claude の設定ファイル（`.claude.json`）を触る部分だ。他の MCP 設定も入っている共有ファイルなので、編集前に必ずバックアップを取るようにした（実際に一度マージ事故が起きたが、すぐ復旧できた）。実行パスも `python` ではなく絶対パスを埋め込み、システムの Python バージョンが変わっても壊れないようにした。

| 詰まり | 解決 |
|---|---|
| PEP 668 で `pip install` が拒否される | pipx に切り替え |
| Python アップグレード後に MCP が出ない | コマンドを絶対パスに |
| 設定のマージで他の項目を消した | 編集前にバックアップを強制 |
| LLM がショートカットツールを呼ばない | description にキーワードを追加 |

## まとめ

MCP サーバーを作ること自体は意外と簡単だ（Python 400 行ほど）。難しいのは *配布*、*キー管理*、そして *LLM がうまく選んで使えるようにする description のチューニング*——この三つだ。先に詰めておけば数日で終わる。

必要な人のためにコードは公開しておいた（[github.com/junstellar/redmine-mcp-jun](https://github.com/junstellar/redmine-mcp-jun)、MIT）ので、もらって使ってもいい。それでも一度は自分で作ってみるのをおすすめする。新しい課題が登録されたら知らせてくれる通知や、入ってきた課題を自動で分析して返信コメントを付けるボットのように——自分に必要なものを自分で組んでみると、はるかに多くが身につく。
